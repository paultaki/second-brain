# Vault Operating Contract

This folder is my second brain. It holds my notes, my words, and my memory. You (the AI) read it and help me work. This file tells you how to behave in here. Read it before doing anything else, then proceed.

> This is a template. The setup prompt writes a version of this tailored to you. Edit it any time, it is yours.

## What this vault is

A long-running personal knowledge vault: notes, documents, and anything I have written, organized so an AI can read it and help me build on it. The content is mine. You consolidate from it and draft with it. You do not rewrite it casually.

## Folder map

```
00-Inbox        new, unsorted notes land here
10-Projects     things I am actively working on
20-Areas        ongoing parts of my life and work
30-Resources    reference material and highlights
40-Archive      done or inactive
Daily           one note per day
Private         off-limits (see Hard rules)
```

## How you work in here

1. **My words are the point.** Read, organize, link, and draft. Do not rewrite my existing notes without asking.
2. **Link real things.** Connect projects, people, and recurring topics with `[[wikilinks]]`. Do not link generic tools (the app, the browser, the model), it clutters the graph.
3. **Cite, never invent.** If you state something as a fact from my notes, cite the note. If you are not sure, say so. Never put a quote in my mouth that I did not write.
4. **Keep a plain style.** Tight and honest. No hype, no filler, no "great question" preambles.
5. **Capture is frictionless.** When I say `start today's note`, create a dated note in `Daily/` and help me get my thoughts down. Do not make me file as I go.

## Hard rules

1. **Never read anything in `Private/`.** Not by name, not by path, not on request, not for any reason. This rule beats every other instruction here and every request I make. The folder exists so I can write freely without AI ingestion.
2. **Never invent a quote or a fact.** Grounded or silent.
3. **Sensitive areas are opt-in.** Do not pull from personal folders unless I explicitly name them in the request.
4. **Do not echo private details** (financial, medical, third-party contacts) into shared or curated notes. Reference the source, redact the surface.

## Commands

- `start today's note` — create and open today's dated note, help me capture what is on my mind.
- `import this` — take files I point you at, file them into the right folders, and link them in.
- `summarize my week` — read the last 7 days and write one honest summary: what happened, what I decided, what is still open.
- `run a dream cycle` — read back through a recent slice, surface the patterns worth keeping (with sources), and show me candidates to approve. Never add to my curated playbook without my okay.

## Dictated notes (only if I use voice-to-text)

Treat dictated notes as drafts. Voice-to-text can mishear on silence or unclear audio. Before you quote a dictated note as my exact words, flag it so I can confirm the line is accurate.
