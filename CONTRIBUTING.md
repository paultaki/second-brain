# Contributing

Thanks for being here. This project is small on purpose, but better ideas are always welcome.

## Ways to help

- **Improve the prompt.** If you found wording that makes the Q&A clearer, the build more reliable, or the result better across different setups, open a PR on [`PROMPT.md`](PROMPT.md).
- **Improve the docs.** Fix anything confusing, wrong, or missing in `README.md` or `docs/`.
- **Share what worked.** Different OS, different note app, an interesting variation, open an issue and tell us. Real-world reports make the prompt more robust for everyone.
- **Report a rough edge.** If the prompt got stuck or did something surprising, an [issue](../../issues) with what you ran and what happened helps a lot.

## Principles to keep

This repo earns trust by staying simple and safe. PRs that hold the line on these are easy to merge:

- **Simple over clever.** The whole thing should stay understandable in one sitting.
- **Setup-agnostic.** It must keep working on Mac, Windows, and Linux, with or without Obsidian, with or without a voice tool. Do not assume one setup.
- **Privacy-first.** Never weaken the `Private`-folder rule, the "no invented quotes" rule, or the local-only, no-telemetry design.
- **Plain language.** Docs and the prompt are written for non-engineers. Keep jargon out.

## How to submit

1. Fork the repo and make your change on a branch.
2. Keep PRs focused, one improvement per PR is easiest to review.
3. In the description, say what setup you tested it on.
4. Open the PR. That is it.

No build step, no tests to run, it is prose and a prompt. Clarity is the bar.
